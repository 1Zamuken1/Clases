1. ADAPTER

Req: DIP
Concepto

Convierte la interfaz de una clase (Librería Externa) en otra que el cliente (Dominio) espera.

En Hexagonal

Es el puente. Implementa el Puerto (interfaz de dominio) usando la tecnología externa. 

// ❌ PAQUETE: com.myapp.service
import com.thirdparty.lib.PdfBoxLibrary;

class ReportService {
    void generateReport() {
        // ALTO ACOPLAMIENTO: Dependencia directa
        PdfBoxLibrary lib = new PdfBoxLibrary();
        lib.renderPage("Reporte...");
    }
}
// 1. PUERTO (DOMINIO): Sin dependencias externas
package com.myapp.domain.ports;
interface PdfPort { void createPdf(String content); }

// 2. ADAPTADOR (INFRAESTRUCTURA)
package com.myapp.infra.adapters;
class PdfBoxAdapter implements PdfPort {
    private ExternalLib lib = new ExternalLib();
    
    public void createPdf(String content) {
        lib.specificMethodCall(content); // Adaptación
    }
}

2. REPOSITORY

Req: SRP, DIP
Concepto

Abstrae el acceso a datos como si fuera una colección en memoria. Oculta la complejidad de SQL o APIs externas.

En Hexagonal

Es un Puerto de Salida. El dominio define la interfaz (el QUÉ guardar) y la infraestructura provee la implementación (el CÓMO en MySQL). 

// 1. PUERTO (Dominio) -> com.myapp.domain.ports
import java.util.Optional;
import java.util.UUID;

public interface UserRepository {
    Optional<User> findById(UUID id);
    void save(User user);
}

3. BUILDER

Req: Inmutabilidad
Concepto

Separa la construcción de un objeto complejo de su representación final. Permite crear objetos paso a paso validando su estado.

En Hexagonal

Fundamental para crear Entidades de Dominio inmutables que siempre nacen en un estado válido. Previene objetos "a medias". 

Order order = Order.builder()
    .withId(UUID.randomUUID().toString())
    .withItem("Pizza Napolitana")
    .withPrice(15.99)
    .build(); // .build() valida reglas de dominio antes de retornar el objeto final
